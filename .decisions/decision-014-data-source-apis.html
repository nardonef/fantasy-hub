Decision 14 of 19

Data Source APIs

Technical

You need four ingestion jobs pulling from Reddit, Twitter/X, NFL stats, and consensus rankings. But the API landscape for each is messy: Reddit's official API now has rate limits and costs, Twitter/X's API is expensive, NFL stats sources range from free-but-fragile to paid-and-reliable, and rankings may require scraping. This decision is about picking a concrete API stack that balances cost, reliability, and data quality. Your constraint is "cost-conscious" so we want to minimize paid API commitments early.

A curated free stack that prioritizes documented APIs where available and falls back to scraping only where necessary. Reddit uses the official OAuth API (free tier, 100 req/min -- more than enough for twice-daily cron). For expert/social signals, Bluesky's AT Protocol is the primary source (free, open, growing NFL analyst community), with Nitter/RSS bridges as a best-effort supplemental Twitter source. NFL stats come from SportsDataIO's free tier (1,000 calls/month, documented endpoints). FantasyPros rankings get scraped from their public pages -- the one scraping dependency, but consensus rankings pages are structurally stable. Total cost: $0/month. Two documented APIs (Reddit, SportsDataIO), one open protocol (Bluesky), and two best-effort scrapers (Nitter, FantasyPros).

Works well when

  • Zero cost with documented APIs where they matter most (Reddit, stats)
  • Bluesky + Nitter together cover more expert signals than either alone
  • SportsDataIO free tier is stable, documented, and sufficient for weekly pulls
  • FantasyPros scraping is a proven pattern -- rankings pages rarely change structure

Watch out for

  • Nitter instances can be unreliable -- treat as supplemental, not primary
  • Bluesky NFL analyst coverage is growing but not yet comprehensive
