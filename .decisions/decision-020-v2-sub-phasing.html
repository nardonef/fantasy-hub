Decision 20 of 25

V2 Sub-Phasing & Build Order

Technical

V2 has two major workstreams: AI Chat (the marquee feature) and Social (recaps, push notifications, profile sharing, manager identity merging). Some pieces have hard dependencies — pgvector and the embedding pipeline must exist before the AI chat can work — but social features are mostly independent. This decision defines whether to build them sequentially with intermediate ship points, or together as one release.

Ship V2 Complete
Build the AI infrastructure and chat experience first — it's the defining V2 feature and has the most architectural unknowns. Social features follow once the AI is working. The 2b milestone gives you something meaningfully different to use before all of V2 is done. Weekly recaps can reuse the LLM infrastructure you built for chat, so social phase benefits from AI work being done first.

Works well when

  • AI chat is the primary reason to build V2
  • Weekly recaps reuse the LLM already set up for chat
  • You want an intermediate shippable milestone

Watch out for

  • LLM provider and cost decisions must be made up front
  • Social features deferred — push notifications ship later

↓
Ship 2a — V1 + Social layer
↓
Phase 2b AI Infrastructure pgvector · LLM SDK · embedding pipeline

↓
Ship V2 Complete
Knock out the social layer first — push notifications, profile sharing, and manager merging have no LLM dependency and are mostly straightforward to build. Weekly recaps need an LLM, but a simpler template-based approach can ship without the full RAG infrastructure. Defers the hardest architectural work (pgvector, RAG pipeline) until the social layer is validated.

Works well when

  • Push notifications and recaps feel more valuable than open-ended chat
  • You want to delay LLM cost/provider decisions
  • Social features are lower-risk and faster to ship

Watch out for

  • AI chat — the V2 marquee feature — ships last
  • Weekly recaps still need some LLM work, so you can't fully defer

Option C
Chosen

Ship Together

Phase 2 — Everything All V2 Features pgvector · AI chat · weekly recaps · push · profile sharing · manager merging
↓
Ship V2 Complete (all at once)
Build all V2 features in dependency order without formal sub-phases, and ship everything together as a complete V2 release. No intermediate milestones — you naturally build infra before AI, and social features whenever. Works if you want V2 to feel like a complete, polished release rather than a trickle of partial updates, and there's no pressure to ship anything early.

Works well when

  • V2 should feel like a cohesive, complete release
  • No external pressure to ship partial features
  • Simplest planning — one milestone, no phase gates

Watch out for

  • Long stretch with no usable new features
  • One blocker (LLM cost, push infra) can stall everything

After a shared infrastructure phase, AI Chat and Social run as independent parallel tracks. This is realistic if you context-switch between them (e.g., do all AI backend work, then all social iOS work), or if you think the work naturally separates cleanly. In practice, one track will finish before the other — the question is just whether you formalize that as a ship point or wait.

Works well when

  • The two tracks genuinely don't block each other
  • You want to context-switch rather than stay in one area

Watch out for

  • In practice, just sequential with extra naming overhead
  • Weekly recaps need the LLM, so tracks aren't fully independent
